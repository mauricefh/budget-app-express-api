## Password Hashing and Salt

Password hashing is the art of encrypting a password. The issue with a simple encryption is the hacker have `rainbow table` which is a list of the commonest used password and their hash value. That where salting a password come in. A salt is a random number of bytes generated by user which we append to a password before encryption and decryption.

Also for hashing it's not recommended to use `SHA256` since it's designed for speed which make it easier to crack. Instead, we should use something like `scrypt` or `bcrypt`

Here is an example database schema

```
User
  - id
  - email (unique)
  - password (hashed, never plain text, will be unique even if 2 user have the same password)
  - salt (random bytes we'll generate must be unique)
```

| id  | email                | password      | salt    |
| --- | -------------------- | ------------- | ------- |
| 1   | user_one@example.com | 482c811da5d5… | fdfs32s |
| 2   | user_two@example.com | 98dssak23sf2… | sak23sf |

## Registration Flow

1. Receive request
2. Validate — is email a valid format? Is password long enough?
3. Check if email already exists in the database
4. Generate salt
5. Hash password with `scrypt`
6. Save user
7. Return success or create session
8. Login user and redirect
